1. What is wrong with these methods?
a) public static double bad (double a, double b) {
    a = a / 2;
    b = b*2;
    return bad(a, b);
}
There is no base case, which will result in a stack overflow.


b) public static int badToo (int n) {
    if (n < 1)
        return 0;
    else if (n==1)
        return 5;
    else
        return 2 * badToo(n+1) + 3;
}

b) The recursive calls do not approach the base case, instead they move
further away from the base case.
